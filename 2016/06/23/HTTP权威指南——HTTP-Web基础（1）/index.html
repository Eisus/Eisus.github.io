HTTP概述

浏览一个页面时（eg.http://www.oreilly.com/index.html），浏览器会向服务器www.oreilly.com发送一条HTTP请求，服务器会去寻找所期望的对象（/index.html），如果成功，就将对象、对象类型、对象长度以及其他一些信息放在HTTP响应中发送给客户端。

Web服务器是web资源的宿主。Web资源是所有能够提供web内容的东西。Web资源可以是静态文件，这些文件可以包含任意内容；资源还可以是跟据需要生成内容的软件。 这些动态内容资源可以根据你的身份、 所请求的信息或每天的不同时段来产生内容。

资源

MIME type：

http会给每种要通过web传输的对象都打上MIME类型的数据格式标签。当web浏览器从服务器取回一个对象时，回去查看相关的MIME类型，看它是否知道应该如何处理这个对象。MIME类型是一种标记文本，表示一种主要的对象类型和一个特定的子类型，中间用一条斜杠分割。

URI

Uniform Resource Identifier，统一资源标识符。URI有两种形式，分别成为URL和URN。现在，几乎所有的URI都是URL。

URL

统一资源定位符。是资源标识符最常见的形式，描述了一台特定服务器上某资源的特定位置。它们可以明确说明如何从一个精确固定的位置获取资源。大部分URL都遵循一种标准格式，包括三部分：

  1. scheme，方案。说明访问资源所使用的协议类型（http：//）
  2. 给出服务器的Internet地址（eisus.github.io）
  3. 其余部分指定了Web服务器上的某个资源 （/about）

URN

统一资源名。URN是作为特定内容的唯一名称使用的，与目前的资源所在地无关。通过URN，还可以用同一个名字通过多种网络访问协议来访问资源。

报文

Web客户端发往Web服务器的HTTP报文称为请求报文（request message）。从服务器发往客户端的报文称为响应报文（response message）。HTTP报文包括三个部分：

起始行

报文的第一行，在请求报文中用来说明要做些什么，在响应报文中说明出现了什么情况。

首部字段

起始行后有零个或多个首部字段，每个首部字段都包含一个名字和一个值，二者之间用冒号来分隔，首部以一个空行结束。

主体

空行后就是可选的报文主体，其中包含了所有类型的数据。请求主体包括了要发送给Web服务器的数据；响应主体中装载了要返回给客户端的数据。注意：起始行和首部都是文本形式且都是结构化的，而主题则是不同，主题中可包含任意的二进制数据。

连接

TCP/IP

HTTP是个应用层的协议。HTTP无需在意网络通信的具体细节，它把联网的细节都交给了通用、可靠的Internet传输协议TCP/IP。TCP/IP提供了无差错的数据传输，按序传输（数据总是会按照发送的顺序到达），未分段的数据流（可在任意时刻以任意尺寸将数据发送出去）。

连接、IP地址及端口号

在HTTP客户端向服务器发送报文前，需要用网际协议（Internet protocol，IP）地址和端口号在客户端和服务器间建立一条TCP/IP连接。在TCP中，需要知道服务器的IP地址，以及与服务器上运行的特定软件相关的TCP端口号（通过URL）。基本的浏览器连接处理：

协议版本

HTTP应用程序要尽量处理不同的HTTP协议变体。目前仍在使用的版本：

  1. HTTP/0.9: HTTP的1991原型版本，这个协议有很多设计缺陷。只支持GET方法，不支持MIME Type、各种HTTP首部或版本号。
  2. HTTP/1.0: 第一个得到广泛应用的HTTP版本。HTTP/1.0添加了版本号、各种HTTP首部、一些额外的方法，以及对多媒体对象的处理。使包含生动图片的Web页面和交互式表格成为可能。
  3. HTTP/1.0+: 非正式的HTTP扩展版本。 添加了很多特性，包括持久的keep-alive连接、虚拟主机支持，以及代理连接支持。
  4. HTTP/1.1: 重点关注的是校正HTTP设计中的结构性缺陷，明确语义，引入重要的性能诱惑措施。是当前使用的HTTP版本。
  5. HTTP-NG (HTTP/2.0): 重点关注的是性能的大幅优化，以及更强大的服务逻辑远程执行框架。

Web的结构组件

代理

位于客户端和服务器之间的HTTP中间实体。HTTP代理服务器是Web安全、应用集成以及性能优化的重要组成模块。

缓存

HTTP的仓库，使常使用网页的副本可以保存在离客户端更近的地方。Web cache/proxy cache 是一种特殊的HTTP代理服务器。可以将经过传送的常用文档复制保存起来。客户端从附近的缓存下载文档会比远程Web服务器下载快得多。

网关

连接其它应程序的特殊的Web服务器，作为其他服务器的中间实体使用。通常用于将HTTP流量转换成其他的协议。

隧道

建立起来后，就会在两条连接之间对原始数据进行盲转发的应用程序，转发时不会窥视数据。

Agent代理

发起自动HTTP请求的半智能Web客户端。eg. spiders，web robots。
