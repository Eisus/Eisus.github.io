响应式网站开发经验总结

发表于 2016-09-14   |   分类于 Tech   |     |   view.

最近跟着幕课网上学完了响应式网站的开发，在学习和自己写的过程中有些经验挺有用的，特此记录下来。

head

PC端样式设计

CSS resets vs Normalize.css

每个浏览器解释html时会有默认样式。统一差异。
resets 会取消所有元素的性质。
Normalize.css 比较先进。优点： 保护了有价值的默认样式，统一了各个样式的展示。同时修复了常见pc端和移动端的bug，比起css resets是一个进步。

单位

  1. px
    固定单位，不会因为页面大小改变。响应式布局中，更多的是用相对长度单位

  2. em
    相对参照物的父元素font-size，找不到父元素的设置，会一直向上。始终找不到，会使用浏览器默认值 1em=16px
    存在问题: 因为相对父元素，容易混乱。会带来很大不确定性

  3. rem
    相对参照物是根元素 html。html没有设置时，会使用 1rem=16px 比较好计算。
    优点：比em安全
    缺点：不是所有浏览器支持rem

看浏览器是否支持某一种样式时可以使用这个网站~~ caniuse

CSS Sprites

提高网页显示速度最有效的一个方法是减少页面的HTTP请求次数，为了减少HTTP请求次数，最直接有效的方法是使用精灵图片（CSS sprites），精灵图片是把许多图片放到一张大图片里面，通过CSS来显示图片的一部分。

于是乎将页面中的背景图整合到一起，利用“background-image”，“background- repeat”，“background-position”的组合进行背景定位的技术被广泛使用与了页面构建中，这就是CSS Sprites。

利用background-position定位，在设置一下宽高，精确定位出背景图片位置。这样在访问一些小的图标时，就不用每个都发起一次请求。–>减少服务器压力，防止图片混乱加载的情况 。

一般使用多个span显示小的图片。首先为span定义普遍属性，background 并使得background-repeat: no-repeat; overflow: hidden; 然后为每一个小的图片既span 定义相应的宽度，高度，和background-position: -45px -45px; 这句的意思相当于把大的背景图先向左移动，再向上移动，然后在左上角抠出固定大小。

透明值

rgba 最后一个参数设置透明度。只会让背景颜色透明度改变，内容不会
使用opacity会让背景和内容的透明度都改变
