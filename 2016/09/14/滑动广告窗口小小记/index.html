发表于 2016-09-14   |   分类于 Tech   |     |   view.

广告滚动组件

一个好的广告滚动组件支持：

  • 不同的图片数量
  • 支持响应式布局
  • 具有良好的兼容性

怎样挑选第三方组件

  • 使用人数
  • 是否开源
  • 文档是否齐全
  • 活跃性 （是否积极维护）
  • 小巧够用的的组件

选组件的时候要注意看组件的兼容性，是否跟网站本身的兼容性一致，要求组件兼容性大于网站

OwalCarousel2

依赖于两个库 jQuery 和 Zepto . Zepto相当于是移动端的jQuery，抛开了pc浏览器的兼容性支持，zepto变得很小巧

关于jQuery， 两个系列版本 1.x 2.x API 基本一样， 2.X 刨除了对老旧浏览器的支持，不再支持IE8一下版本的浏览器，变得更轻巧，使用2.x 对于国内浏览器偏激进。

要注意，把oc 的css引用都放在了head上
把脚本的引用都放在了body结束，这样就有效的防止了在页面引入速度变慢时，因为脚本的阻塞变成一片空白。

还要注意，oc 中theme.default.min.css是需要自定义的。可通过官方网站老版本获得，也可通过csdn.js获得 （js库，通过他来访问库）

响应式图片

图片的排版和布局+根据设备大小加载不同的图片。方式包括：

js或服务端 （控制图片的加载）

使得 window在加载和改变尺寸的时候，调用该方法。或者把屏幕或设备信息写入cookie，在服务器端决定返回哪种尺寸图片.
优点： 兼容性较好
缺点： 修改时不方便

srcset
是img标签里的一个属性。

浏览器会根据页面宽度选择图片。 在大的图片加载后，把界面缩小，浏览器还是会默认使用大的图片，不会使用小图片。

当dpr=1时，就按设置的临界点选择加载哪个图片。当dpr=2时，并不简单的使用像素值/dpr, 要比临界值高一点。 因为浏览器会综合各种因素，包括设备或当时的网速等
缺点： 当把content设为50%时，屏幕变为480时图片改变，即图片只有240时就改变了。

srcset配合sizes

100vw代表视口的宽度,100代表百分比。

picture

浏览器会遍历picture里的source设置，直到找到一个属性满足当前环境。实现多组条件的匹配
