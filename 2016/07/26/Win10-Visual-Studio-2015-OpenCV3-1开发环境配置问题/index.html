Win10+Visual Studio 2015+OpenCV3.1开发环境配置问题

发表于 2016-07-26   |   分类于 Tech   |     |   view.

新电脑重新配置visual studio 2015 + opencv 3.0，虽然参考了大量教程，但依旧遇到了各种奇奇怪怪的问题，记录一下，以供参考。
配置教程网上很多，不再赘述，主要参考教程有：

OpenCV 学习笔记1 — 开发环境配置（win10+VS2015+OpenCV2.4.12）
Setting up OpenCV 3.1 in Visual Studio 2015

以下是遇到的主要问题和解决方法：


遇到这个问题首先检查的是环境变量是否配的正确，配错了会导致dll文件无法找到。一般配对了就不会有这样的问题产生。环境变量配对了但依旧有dll文件无法找到的问题，可以考虑自己下载相应文件，然后将其复制到相关目录下：
32位系统：C:\Windows\System32
64位系统：C:\Windows\SysWOW64

如果问题还未解决，微软官方给的解决方案是**安装microsoft redistributable**，我有尝试，但我的问题并未得到解决，但不失为一种办法可以尝试一下。


遇到类似问题，可能是和opencv版本有关系的。之前配opencv2.4.13一直出现这个问题，换了opencv3.1之后这个问题就没有了。如果换了版本问题没有得到解决，可以尝试以下方案：
方案一：
Step 1. Open Windows Explorer and navigate to C:\Windows\System32. Step 2: Locate and delete the following files:
